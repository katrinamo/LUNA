User Scenarios

  1. Accessing the App and Logging in for a first time user

    1. User goes to the online consent form at the following link: https://luna-app.000webhostapp.com

    2. User reads the consent form and enters in their birthday. If they agree to the study then the user presses the “I consent” button on the website.

    3. If the user is between the ages of 18-25 they will be allowed to download the app and enter into the study. They will then be taken to a page that asks them to provide their UK email address. They will provide this and then press the “register” button. If the user provided a UK email address (ends in @uky.edu or @g.uky.edu), the user continues with step 4. If not a UK email address, an error message is displayed.

    4. The page displays where to download the app. Additionally, an email will be sent to their UK email account. This email will contain a random string that must be used to access the LUNA App once downloaded.

    5. The user will then go to the site where they were told to download the app and download it.

    6. Once the app is downloaded onto their mobile device the user can access the app.

    7. The app will open in a login page and the user will be asked to enter in the random string sent to their UK email account. The user will also be asked to create a username and password that are both 8 characters or longer.

    8. If the random string entered in is correct, and the username and password supplied are valid, then the user will be allowed access to the app and an account will be created for them.

  2. Answering the OnBoarding Questions

    1. After logging in as a first-time user, the user will then be taken to the onboarding questions page within the app. This page contains a series of 9 questions that are essential for the study and for proper app execution.

    2. The user will answer the onboarding questions by sliding from question to question within the onboarding questions page. The questions consist of the following:

      1. Birthday

      2. Cycle length

      3. Period length

      4. Birth control

      5. Reproductive disorder

      6. Last period

      7. Relationship status

      8. Time of the day to input responses to daily questions

      9. Intention of whether to become pregnant.

    3. The user will then press the submit button on the final slide of the onboarding questions page of the app. If all the questions are properly answered then the responses will be sent to the server and the user’s settings will be saved. If a question is left unanswered then an error message alert will be output to the app screen.

    4. The app will then go to the main home page of the app which consists of 3 major tabs.

  3. Daily App Operation

    1. The app can be closed and the mobile device can be powered off but if the app remains on the mobile device and isn’t deleted then the user will never be logged out of their account. The user will never again have access to the original login or onboarding questions pages as they are no longer necessary.

    2. When the app is opened it will jump to the home page. This home page will display a calendar and will consist of 3 major app tabs named “Calendar”, “Tracker”, and “Settings”.

    3. To access a specific tab/page within the app a user simply must tap on the tab they want access to.

    4. The major tabs of the app have the following features:

      1. Calendar Tab: The calendar tab will display a calendar of the current month. For a user to change months they simply must slide the calendar left or right. Eventually the calendar will have markers on it indicating when a user’s cycle is starting and ending. However, this feature has not yet been added to the app and will be a future enhancement.

      2. Tracker Tab: The tracker tab will allow the user to enter in their responses to the daily questions. Additionally, a future enhancement to the tracker tab will be to allow the user to access their past responses to their daily questions to view how their responses are changing over time.

      3. Settings Tab: The settings tab will allow the user to view and modify their original or latest responses to the onboarding questions. To access this page and change settings a user must first enter in their username and password.

  4. Answering Daily Questions

    1. To respond to the daily questions a user will first click on the “Tracker” tab at the bottom of the main page of the app.

    2. The user will then select the “Daily Questions” option within the “Tracker” page of the app.

    3. The user will then be asked 3 questions:

      1. If they are currently on their period

      2. Their level or desire of sexual interest

      3. If they have engaged in sexual intercourse today

    4. If the user responds “no” to the 3rd question then they can submit their answers to the server by hitting the “submit” button and their responses will be sent if every question is answered properly.

    5. If the user responds “yes” to the 3rd question then they will be given more questions to answer. The user will then be asked 9 more questions:

      1. How often they have engaged in sexual activity or intercourse today

      2. How satisfied they were with the emotional closeness they felt during sexual activity with their partner

      3. How satisfied they are with their sexual relationship with their partner

      4. How satisfied they are with their overall sexual life

      5. Their level of sexual arousal or turn on during sexual activity

      6. How confident they were about becoming sexually aroused during sexual activity

      7. How difficult it was to become lubricated during sexual activity

      8. How difficult it was to maintain lubrication until completion during sexual activity

      9. If they had sexual stimulation or intercourse

    6. If the user responds “no” to the 9th question then they can submit their answers to the server by selecting the “submit” button and their responses will be sent if every question is answered properly.

    7. If the user responds “yes” to the 9th question then they will be given more questions to answer. The user will then be asked 3 more questions:

      1. How difficult was it to reach orgasm

      2. How satisfied they were with their ability to reach orgasm.

      3. If they had intercourse

    8. If the user responds “no” to the 3rd question then they can submit their answers to the server by selecting the “submit” button and their responses will be sent if every question is answered properly.

    9. If the user responds “yes” to the 3rd question then they will be given 1 more question to answer. The user will be asked this final question:

      1. To rate their level or degree of discomfort or pain during intercourse.

    10. The user’s responses will then be sent to the server once the “submit” button is selected if every question is answered properly.

  5. Accessing the Settings Page and Changing Settings

    1. To access the settings page a user must first enter in the username and password they created on the original login page

    2. If the username and password entered in are correct then the app goes to the settings page. If the username or password entered are incorrect then an error message alert is output to the screen.

    3. On the settings page, a user can view their initial/most recent responses to the onboarding questions. The user also has the option to update their responses to the following questions:

      1. Birth Control

      2. Relationship Status

      3. Time of the day to input responses to the daily questions

      4. Intention on whether to become pregnant

      5. Reproductive Disorder

    4. To change a response to an onboarding question a user must toggle the button corresponding to the question they want to update, make the update, and then submit their new answer to the question.

    5. Once one or more question responses have been updated, the user has the option to submit their updated responses to the server. Their new settings are then sent to and saved server side.
